Dev Notes

Multi-party rewrite — 2026-06-11

The app was redesigned from a 1-to-1 "I speak / They speak" pair into a multi-party listen-only translator. Each participant runs the app on their own laptop, picks the language they want to hear, and the app translates everything coming through Teams audio into that language. If a speaker is already speaking the listener's language, no translation is produced.

Key architecture changes

  1. One Realtime session per client, not two. The "outgoing" mic path is removed entirely — your own voice is translated by everyone else's app.
  2. Whisper auto-detects language per turn. The session prompt tells the model to translate to a single target and to stay silent when input is already in target.
  3. Single chronological transcript pane replaces the two-column "What I Say / What They Say" layout.
  4. Feedback-loop suppression in useAudioCapture: system-audio chunks are dropped while AudioPlayer is actively playing translated output. Breaks the Windows render-loopback → capture → retranslate echo.

Known limitations

  • Audio mixing is naive — user hears the original Teams audio AND the translated overlay simultaneously. Use Teams' captions-only or reduce Teams volume during demos.
  • Same-language detection is fully delegated to the model. If it doesn't follow the "stay silent" instruction perfectly you may occasionally get a redundant English-to-English passthrough.
  • The "turn_skipped" event is inferred from response.done producing no audio output — works in practice but not provably airtight.

Earlier Windows packaging fixes (still relevant for next builds)

  • Server's runtime node_modules must be bundled into resources/server/ by electron-builder.
  • @orka/shared's main field must point at the built file, with explicit named re-exports (TS's __exportStar isn't statically detectable by Node's CJS lexer).
  • waitForServer retry budget bumped from 30 to 120 for cold-start.
  • .env shipped without NODE_ENV=development; the production default in main.js wins.
Setup Guide — Nokia Personnel

Prerequisites

  • Windows 10 or 11 (x64).
  • An OpenAI API key with access to the Realtime API. Ask Pratyada if you don't have one.
  • No Node.js install required — the runtime is bundled.

1. Get the app

  1. Copy the installer/zip provided by Pratyada to your laptop.
  2. Extract or run the installer.

2. Configure your API key

  1. Locate the app folder (next to Nokia Orka Translator.exe) and open .env in Notepad.
  2. Replace the value after OPENAI_API_KEY= with your own key.
  3. Save and close.
OPENAI_API_KEY=sk-proj-...your-key-here...
ANTHROPIC_API_KEY=
DEEPGRAM_API_KEY=
PORT=3001
NODE_ENV=production

3. Run it

  1. Double-click Nokia Orka Translator.exe.
  2. First launch takes ~5–10 seconds to start the embedded server.
  3. Allow microphone / screen-share access when Windows prompts (needed for system-audio capture).
  4. The app pins to the system tray. Right-click to quit fully.

4. Using it in a multi-party meeting

  • Each participant runs the app on their own laptop and picks the language they want to hear.
  • Pick your target from the "I want to hear in" dropdown.
  • Click Start Listening. The app starts capturing Teams audio.
  • Whenever someone speaks a language different from yours, you'll see the original transcript and hear the translation in your speakers/headphones.
  • When someone speaks your language, the row is marked "already in your language — no translation needed" and stays silent.
  • Use headphones if you can — reduces audio bleed between Teams and the translator.

5. Troubleshooting

  • No translation appearing: check your OpenAI key and network. Verify NODE_ENV=production in .env.
  • "Cannot find server" error: the resources/server/ folder is corrupted — reinstall.
  • Port 3001 in use: change PORT=3001 in .env and relaunch.
  • Antivirus quarantine: Nokia endpoints sometimes block unsigned Electron exes. Add an exclusion for the app folder.

6. Sharing with another colleague

Share the installer/zip Pratyada gave you. The recipient extracts, edits .env with their own key, and double-clicks the exe.
